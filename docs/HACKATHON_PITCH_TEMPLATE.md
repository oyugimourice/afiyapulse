# AfiyaPulse Hackathon Pitch Template

## Recommended Timing

Target length: 4 to 6 minutes.

Spend less time explaining healthcare generally and more time showing the product. Judges reward clarity, demo readiness, and a believable path forward.

## Slide Flow

1. Title
   - Say: "AfiyaPulse turns clinical conversations into structured documentation."
   - Goal: Make the product category obvious in 10 seconds.

2. The Problem
   - Say: "Clinicians lose time after visits turning conversations into notes, prescriptions, referrals, and follow-up plans."
   - Keep it grounded. Avoid overquoting big statistics unless you can cite them.

3. Our Solution
   - Say: "AfiyaPulse is not just transcription. It supports the full workflow from patient context to clinician-approved documentation."
   - Emphasize human review.

4. Live Demo Flow
   - This is your transition into the app demo.
   - Show: login, dashboard, patients, consultation, review panel.

5. How It Works
   - Explain the pipeline: capture, transcribe, structure, draft, review.
   - Say: "AI accelerates the work; clinicians stay accountable."

6. What We Built
   - Highlight actual implementation:
   - React + TypeScript frontend
   - Express API
   - Prisma data model
   - Redis/WebSockets
   - AI service layer and clinical agents

7. Why It Wins
   - Tie judging criteria together:
   - Real problem
   - Working prototype
   - Technical depth
   - Responsible AI
   - Clear healthcare impact

8. Roadmap
   - Talk about the next 30 to 90 days:
   - Better transcription reliability
   - Specialty templates
   - Pilot-ready deployment
   - FHIR/EHR export

9. Impact Potential
   - Mention what you would measure:
   - Minutes saved per consultation
   - Draft acceptance rate
   - Documentation completion time
   - Clinician satisfaction

10. The Ask

 - For a hackathon, ask for pilots, mentorship, clinical feedback, and integration support.
 - Close with: "AfiyaPulse helps clinicians spend less time documenting care and more time delivering it."

## Demo Checklist

- Start the API on port 3001.
- Start the web app on port 3000.
- Log in before presenting if possible.
- Have the dashboard open in one tab.
- Have `/patients` open in another tab.
- Prepare one sample patient.
- Prepare a short consultation scenario.
- Avoid live typing long clinical text during the demo.

## Strong Closing Line

"Healthcare documentation should not compete with patient care. AfiyaPulse brings AI into the workflow carefully: it drafts, the clinician decides, and the patient gets better continuity of care."
